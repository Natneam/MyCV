EM, REM and % units

Before choosing which one is the best for responsiveness of websites lets get know what em, rem, % size units are.

EM

An EM is a unit of typography, equal to the currently specified font size. The default size of the root(html) font size is 16px unless specified by the user. This means by default, when we say 1em it means 16px.

Lets look at the CSS styles below

h1{
     font-size:1em; /* This means the font size is 16px*/
         margin : 2em ; /* This means 2*16px which is 32px*/
}

h1{
    font-size:2em; /* This means 2*16px = 32px (current font size) */
    margin:2em; /* This means 2*32px = 64px*/
}

In the first h1 styling code the font size is 1em which is 16px(the default font size as specified in the browser setting) after wards If we style using em, 1em value is 16px unless we changed the font size. While in the second h1 styling code the the original font size were 16px but when we change the font size to 2em the current font size is no longer 16px but 32px after wards if we style using em, 1em will be 32px.

The em unit is confusing because the value of em could be different in different part of our code.

REM

An REM is a unit of typography, equal to the root specified font size. When we specify size using rem it means we are changing the size of the element using the factor of the root font size. In html documents the root is <html> tag. The end user can change the default font size of the root(<html>) in the browser setting.

Lets see the following CSS code snippets

h1{
    font-size:1rem; /* This means the font size is 16px*/
    margin : 2rem ; /* This means 2*16px which is 32 px*/
}

but

h1{
    font-size:2rem; /* This means 2*16px = 32px */
    margin:2rem; /* This means 2*16px = 32px */
}


In the first code snippet the font size is 16px because it is 1*16px = 16px. The margin is 32px because the root font size is 16px then 2rem = 2 * 16px. Where as in The second code snippet the font size is 32px because 2*16px = 32px. The margin is also 32px because 2rem = 2 * 16px = 32px.

percentage (%)

Percentage unit defines a measurement as a percentage relative to an enclosing element.

Lets see the following code snippets

HTML

<div>
    <p> Hello world<p>
<div>


CSS

div{
    width : 400px;
}

p{
    width : 80%;
}


In the html code the <p> is enclosed in <div> so when we say the width of <p> is 80% it means <p> will take 80% of the <div> which is 400*0.8 = 320px.

It is not recommended to use percentage unit to specify font size.


In my opinion, using rem for size specification is more suitable because it will save us from the confusion of different em values on different part of our code. Sometime it is plausible to use percentage unit for width, margin and padding size specifications. Using em, rem or percentage will make our website responsive (in terms of measurement but not in terms of orientation and automatic element placements) because all are relative to the device screen size and the browser setting. But we have to be cautious when we use them.

What does it means 'the default browser font size is 16px'

When we say The default browser font size is 16px. It means unless we change it in the setting or we zoom, 16px is the size of ordinary texts. When we change it to 200px the default font size will be 200px. Changing the default font size will result effect on em and rem units to as specified in the previous section.

Changes In CSS3

Mobile-first mentality
Mobile-first mentality is starting developing the web for mobile devices and then using media queries to change the layout as the screen size changes. Media queries are calls made by the code to determine what device and size screen a user is viewing the site on.

Example:

/* Set the background color of body to green */

body {
      background-color:green;
}

/* On screen grater than or equal to 600px, change the background color to red */

@media screen and (min-width: 600px) {
   body {
        background-color:red;
   }
}

/* On screen grater than or equal to 800px, change the background color to yellow */

@media screen and (min-width: 800px) {
   body {
        background-color:yellow;
   }
}


Selectors
In previous version of CSS, we can edit HTML elements property by using their tag name, class or id individually or by listing the hierarchies but in CSS3 it provide a wide variety of selectors to use like

[attr^=val]- matches a HTML element with the attribute attr and a value starting with val
[attr$=val]- matches a HTML element with the attribute attr and a value ending with val
[attr*=val]- matches a HTML element with the attribute attr and a value containing the substring val

Backgrounds and borders: CSS3 enables rounded corners on boxes and drop shadows. Before CSS3, backgrounds had to be achieved with images, which added to a page’s file size and load time. Text effects - shadow effects, text overflow, word breaking (automatically breaking text so it fits within a box), and text wrapping.
Web font support
It enables to run web fonts from Google Fonts or Typecast. We can download a font to our website directory or we can directly access from the its source online.
2D and 3D transformations, animations, and transitions
It will allow to us to create animated effects on our web page like rotate, shrink, translate into different colours and the like. Example

div {
transform: rotate(20deg);
}
New colour scales and image effects
CSS3 added new colours scales like RGBA(red,green,blue,alpha), HSLA(hue, saturation, lightness, aplha) and HSL(hue, saturation, lightness). It also added image effects like gradient colours adjustments, rounded image corners (border-radius property) and opacity. Which before requires a lot of work on Photoshop .
Box-sizing
It helps to get the sizing of elements as we want without having to subtract dimensions for padding and borders.
